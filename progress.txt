## Infrastructure observations (for context)

- **Faucet rate-limit behavior.** The 2026-04-19 session hit a "19h16m49s" cooldown even with a fresh random key. This session, switching to a different IP cleared the block instantly — confirming the skill's note that the cooldown is IP-based, not per-address. The persisted-mnemonic flow is therefore worth it: you pay the faucet tax once, then every run deploys from the same funded account.
- **PulseChain v4 testnet gas is trivial.** A full TitrateSimple deploy consumes a small fraction of 1 tPLS on chainId 943, so one 10 tPLS top-up covers many runs. Real gas-sensitive tuning still needs Sepolia (mainnet-cost testnet).
- **Playwright noise filter.** The `shouldIgnore()` filter in `smoke.spec.ts` drops Chromium's "Failed to load resource" console messages so third-party 403s (api.web3modal.org without a Reown project-id allowlisted for `localhost:4173`) don't mask real JS regressions. `pageerror` events + non-network console errors still flow through.
- **Vitest glob and Playwright specs.** Default vitest glob matches `*.spec.ts` and tried to execute Playwright specs, exploding with "test.describe() not expected here." Fix: `exclude: ['node_modules', 'dist', 'e2e/**']` in `vitest.config.ts`. Worth remembering if any sibling workspace adds Playwright later.
- **1Password SSH signer still flakes.** Had to unlock mid-session again. Same pattern as prior checkpoints. Not deterministic; no workaround other than unlock + retry.
- **`gh auth switch -u morbiuscashdev`** still required before most merges/PR creates when the active account reverts to `provexdev`.
