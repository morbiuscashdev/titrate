## Notable non-obvious test patterns (learned the hard way)

Two subtle gotchas tripped the first test runs — worth keeping in mind for future loop work:

- **`bus.emit` is synchronous.** If the driver emits `completed` before its first real `await` (e.g. the non-block-scan first-step branch), any handler registered *after* `loop.start()` will miss it. Fix: `const completedP = new Promise(res => bus.on('completed', res))` BEFORE `loop.start()`. Already used downstream — document if writing more loop tests.
- **`stop()` on a driver parked in `bus.once(...)` will hang.** The stopping flag is set by `stop()`, but the driver is inside a pending Promise and can't observe it until the bus resolves. Fix: emit a wake event (`pipeline-changed` or `scan-progressed`) between `loop.stop()` and awaiting the stopPromise. Pattern used in the filter-loop `stop() mid-run` test.
