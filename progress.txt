## Notable test patterns worth keeping

- **`vi.hoisted` + `vi.mock` + top-level `await import`** — filter/source/distributor unit tests all use this shape to mock cross-module dependencies while keeping a single shared `vi.fn` they can reconfigure per-test.
- **In-memory loop fixtures (`memory-storage.ts`)** — drop-in replacements for `createAppendableCSV` / `createCursorStore` / `createAppendableJSONL`. No tmpdir, no fs, no flakes. Saved as a feedback memory: **test each piece, maximize determinism**.
- **`vi.fn<RunSourceFn>(...)`** preserves the function signature through `mock.calls`, so assertions on `c[1]` type-check correctly. Without the generic, vitest infers `[][]`.
- **`page.exposeBinding` for headless RPC** — eliminates the need to run actual EIP-712 crypto inside Playwright's addInitScript context. Node-side viem handles real signing + broadcasting; browser just forwards.
- **Report chainId X, broadcast on chainId Y** — the `reportChainId` option on the headless wallet lets wagmi see a chain it knows while the underlying viem walletClient signs + broadcasts on a different one. Sidesteps AppKit's "unsupported chain" prompt without adding PulseChain v4 to the prod networks list.
