- **PR #27 `a927589` — test(web): stabilize 2 CI-flaky tests.** FiltersStep `runs preview and shows surviving count` used `getByText(/addresses pass/)` which matched both the pre-preview info banner AND the survivor readout — on slow CI `waitFor` resolved on the banner and `getByText` returned the wrong element. - **PR #28 `526717d` — fix(web): copy storage-campaign package.json into Docker build.** Railway build was failing with `@titrate/storage-campaign@npm:0.0.1: 404 Not Found` because the Dockerfile pre-copied 6 of 7 workspace package.json files, missing `storage-campaign`. TUI's manifest depends on it, so inside the container Yarn didn't recognize it as a workspace and tried the public registry. One-line fix.

### Production audit fixes (PRs #29–#31)

Audited the live Railway deploy via Chrome DevTools MCP, found and fixed five issues:

- **PR #29 `8cdf651` — fix(web): production header nav, CSP, and Lit dev-mode warning.**
  - Header anchors used path URLs (`/`, `/settings`) but App uses `<HashRouter>`. Clicking the gear served `index.html` via Caddy's SPA fallback and the router rendered the landing page. Switched to `#/`, `#/settings`.
  - CSP `connect-src` was missing `https://api.web3modal.org` (where Reown AppKit actually fetches config) — 6 blocked bootstrap requests per page load. Also added `https://cca-lite.coinbase.com` (Coinbase Wallet telemetry).
  - Lit "dev mode" warning: pre-populated `globalThis.litIssuedWarnings` with `["dev-mode"]` in index.html (empty Set seed didn't work — Lit checks `has(warning) || has(code)`, so needed the actual code string).
- **PR #30 `cdfb11e` — fix(web): stop precaching index.html so CSP changes propagate.** After #29 shipped, users were STILL seeing the old CSP because the VitePWA service worker precached the old `index.html`. CSP meta tag is in HTML, so every CSP change was gated behind user clicking Refresh. Fix: `globPatterns` drops `**/*.html`, `navigateFallback: null`, runtimeCaching NetworkFirst for `request.mode === 'navigate'` (3s timeout, 24h offline fallback). Also switched `registerType: 'prompt' → 'autoUpdate'` with `skipWaiting: true` + `clientsClaim: true`. Future deploys propagate within one navigation.
- **PR #31 `5f11d74` — fix(web): tri-state theme toggle, single toggle, unlock contrast, Lit warning.**
  - Double theme toggle on landing page (LandingPage had its own absolutely-positioned `<ThemeToggle />` in addition to the Header's) — removed duplicate.
  - System theme option was lost in the brand migration — the new `ui/ThemeToggle` was light/dark only. Expanded `Theme` to `"light" | "dark" | "system"`, added `ResolvedTheme` + `resolveTheme()`, `useTheme` now returns `{theme, resolvedTheme, setTheme, toggle}` with matchMedia listener when theme is "system". Two-segment button group: left flips light/dark, right locks to system.
  - Settings unlock banner had poor dark-mode contrast (legacy `text-yellow-400` on `bg-yellow-900/20` = orange-on-orange). Switched to brand `--color-warn` tokens with brutalist chrome.
