# Titrate — Progress Checkpoint

## Date: 2026-04-04

## What Was Accomplished

### Phase 1: Contracts + SDK Foundation (complete)
- Monorepo scaffold (npm workspaces, Foundry, Vitest)
- TitrateSimple.sol + TitrateFull.sol with selector-scoped allowance, on-chain registry, multicall
- SDK modules: types, storage interface, chains, CSV, wallet (EIP-712), encode, sets

### Phase 1B: Scanner, Pipeline, Distributor (complete)
- Scanner: block scanning, Transfer event log scanning with dynamic range titration
- Pipeline: composable set operations, serializable configs
- Distributor: deploy, verify, disperse (streaming per-batch LiveFilter), allowance, registry

### Phase 2: TUI (complete)
- Filesystem storage adapter, CLI commands, interactive wizard, e2e tests

### Phase 3A: SDK Extraction + Web Component Library (complete)
- 8 functions extracted from TUI to SDK
- 2 new SDK functions (computeRequirements, aggregateSpendReport)
- Web scaffold: Vite + React 19 + Tailwind v4 + Vitest
- 15 pure React components (layout, data display, forms, feedback)

### Explorer API Scanner (complete)
- ExplorerBus: adaptive per-domain rate limiting (80% on first 429, -5% per subsequent)
- Block range bisection for 10k result cap
- 4 scanner functions: token transfers, transactions, internal txs, balances
- Pipeline integration: 'explorer-scan' source, 'explorer-balance' filter

### TrueBlocks Scanner (complete)
- TrueBlocksClient: auto-pagination via firstRecord/maxRecords
- 5 scanner functions: appearances, transfers, balance history, traces, status
- Integration tests gated by TRUEBLOCKS_URL env var
- Pipeline integration: 'trueblocks-scan' source, 'trueblocks-balance-hint' filter

### Gas Improvements (complete)
- Removed silent 500k gas fallback
- Speed presets (slow/medium/fast) for headroom and priority
- Fee history percentile-based priority fee
- Split base fee cap + priority fee cap, cumulative cost limit
- parseGwei utility, updated TUI flags

### CLI Param Casing (complete)
- All multi-word CLI flags accept both kebab-case and camelCase

### Human Interventions Layer (complete)
- SDK validation: address (hex, length, prefix, checksum, duplicates), amount (negative, zero, large), batch, helpers
- SDK intervention: 8 points, 13 actions, hook/config/journal interfaces, spot check sampling
- TUI: review file CSV write/read, JSONL journal, spot check display, intervention handler orchestrator
- Stuck transaction handling: bump/wait/abort

### Phase B: Distribution MVP (complete)
- SDK: ChainConfigStore, AppSettingsStore added to Storage interface
- storage-idb: chain config and app settings object stores (DB v2 migration)
- Web crypto: AES-GCM encrypt/decrypt with keccak256 key derivation
- Encrypted storage wrapper: field-level encryption (rpcUrl, explorerApiKey, wallet addresses)
- 5 providers: ThemeProvider, WalletProvider (Reown AppKit), StorageProvider, ChainProvider, CampaignProvider
- Step locking: 7-step progressive flow (campaign → addresses → filters → amounts → wallet → requirements → distribute)
- Perry mode: cold wallet derives hot wallet via EIP-712 signature
- 4 query hooks: useTokenMetadata, useNativeBalance, useTokenBalance, useGasEstimate (TanStack Query)
- 3 pages: HomePage (campaign grid), CampaignPage (step flow orchestrator), SettingsPage (chain config CRUD)
- 7 step forms: CampaignStep, AddressesStep, FiltersStep, AmountsStep, WalletStep, RequirementsStep, DistributeStep
- Header with ThemeToggle (light/dark/system), EncryptedField with lock icon
- React Router v7, provider tree, App.tsx wiring
- 3 TS fixes: Uint8Array buffer cast, FilterType narrowing, vitest import

### Phase B: Gap Fixes (complete)
- ChainProvider now wired to active campaign via ChainBridge component
- CampaignProvider moved above ChainProvider in provider tree
- Wallet badge (Reown appkit-button) rendered in Header
- completeStep() API added to CampaignProvider for explicit step completion
- Filters step requires explicit save/skip (no longer auto-completes)
- Vite build: Node builtin shims for SDK barrel export (node:fs, node:path, node:url, node:crypto)
- Dark body background via Tailwind CSS, class-based dark mode configured
